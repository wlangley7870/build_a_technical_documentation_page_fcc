C# Technical Documentation

Introduction

The C# language is the most popular language for the .NET platform, a free, cross-platform, open source development environment. C# programs can run on many different devices, from Internet of Things (IoT) devices to the cloud and everywhere in between. You can write apps for phone, desktop, and laptop computers and servers.

C# is a cross-platform general purpose language that makes developers productive while writing highly performant code. With millions of developers, C# is the most popular .NET language. C# has broad support in the ecosystem and all .NET workloads. Based on object-oriented principles, it incorporates many features from other paradigms, not least functional programming. Low-level features support high-efficiency scenarios without writing unsafe code. Most of the .NET runtime and libraries are written in C#, and advances in C# often benefit all .NET developers.

Prequisites

While there are no strict prerequisites for learning C#, a basic understanding of the following concepts can be helpful:

  • Basic programming logic: Understanding how to break down problems into smaller, solvable steps.
  • Variables and data types: Knowing how to store and manipulate different types of data.
  • Control flow statements: Understanding how to control the flow of execution in a program using if-else statements, loops, and other constructs.
  • Functions and procedures: Knowing how to define and call reusable blocks of code.
  • Prior programming experience is helpful in making the learning process smoother as you'll already be familiar with the above general concepts, allowing you to be up and running with a potentially shorter learning curve.

    Hello World

    Here is a basic introduction to C# per the customary "Hello, World!" program:

                        using System;
                    namespace HelloWorld;
                    {
                    class Program
                    {
                        static void Main(string[] args)
                        {
                        Console.WriteLine("Hello, World!");
                        }
                    }
                    }

    Explanation:

    1. using System;: This line imports the System namespace, which contains fundamental classes like Console
    2. namespace HellowWorld: This declares a namespace to organize your code.
    3. class Program: This defines a class named Program, which is the entry point of the application.
    4. static void Main(string[] args): This is the Main method, the starting point of the program execution.
      • static: This keyword indicates that the method belongs to the class itself, rather than to an instance of the class.
      • void: This means the method doesn't return any value.
      • string[] args: This parameter is an array of strings that can be used to pass arguments to the program from the command line.
    5. Console.WriteLine("Hello, World!");: This line prints the message "Hello, World!" to the console.
      • Console.WriteLine: This method writes a line of text to the console.

    To run this code:

    1. Create a new C# project: Use your chosen IDE (like Visual Studio or Visual Studio Code) to create a new C# console application project.
    2. Paste the code: Copy and paste the code into the Program.cs file of your project.
    3. Run the program: Build and run the project. You should see the output "Hello, World!" in the console.

    • Variables and Data Types

    In C#, variables are used to store data. Before using a variable, you must declare its data type, which specifies the kind of data it can hold. C# is a strongly-typed language, meaning that you must declare the type of a variable before using it.

    Data Types

  • C# categorizes data types into two main groups:
    1. Value Types

      2. Store data directly within the variable itself.

        Integral:

      • sbyte: Stores a signed integer between -128 and 127
      • byte: Stores an unsigned integer between 0 and 255
      • short: Stores a signed integer between -32,768 and 32,767
      • ushort: Stores an unsigned integer between 0 and 65,535
      • int: Stores integer numbers(e.g., 42, -10)
      • uint: Unsigned 32-bit integer (0 to 4,294,967,295)
      • long: Stores a signed integer between -9,223,372,036,854,775,808 and 9,223,372,036,854,775,807
      • ulong: Stores an unsigned integer between 0 and 18,446,744,073,709,551,615

        • Floating-Point:

      • float: Stores single-precision floating-point numbers (e.g. 3.14f)
      • double: Stores double-precision floating-point numbers (e.g., 3.14159265359)
      • decimal: Stores decimal numbers with high precision (e.g., 123.45m)

        • Character:

      • char: Stores a single Unicode character (e.g., 'A', '!', '€')

        • Boolean:

      • bool: Stores a logical value (true or false)

        • DateTime:

      • DateTime: Represents a data and time value;

        Enum:

      • Defines a set of named constants.
      • Example:

        User-defined value types.

    2. Reference Types

      3. Store a reference (memory address) to the data

      • Reference to a method

    Value Types
    • Stored directly on the stack

    Key Considerations

    • Choosing the Right Data Type: Select the data type that best suits the type and range of values you need to represent. This helps to optimize memory usage and prevent potential errors.
    • Type Safety: C# is a type-safe language. This means that you cannot assign a value of one data type to a variable of another incompatible data type without explicit conversion (casting).
    • By understanding these fundamental concepts of variable types, you can effectively write C# code to create various data structures and algorithms.

    Variable Scope

  • Variable Scope refers to the region of a program where a variable is accessuble. In C#, the scope of a variable is determined by where it is declared.

    • Types of Scope in C#

    1. Local Scope:
      • Variables declared within a method, loop, or code block have local scope.
      • They are only accessible wihin that specific block.
      • Once the block ends, the variable is destroyed
                              void MyMethod()
                        {
                        int x = 10; // Local to MyMethod
                        //...
                        }
    2. Class-Level Scope:
      • Variables declared within a class but outside any method have class-level scope.
      • They are accessible to all methods within that class.
                                  Class MyClass
                            {
                              int classVariable = 20; // Accessible to all methods in MyClass
                              
                              void Method1()
                              {
                                Console.WriteLine(classVariable);
                              }
                              
                              void Method2() {
                                Console.WriteLine(classVariable);
                              }
                            }
    3. Namespace Scope:
      • Variables declared within a namespace are accessible to all classes within that namespace.
      • This is less common in practice
    4. Static Scope:
      • Variables declared with the static keyword belong to the class itself, not to specific instances.
      • They can be accessed without creating an instance of the class
        •                             class MyClass
                            {
                              public static int staticVariable = 30;

                              public static voide MyMethod()
                              {
                                Console.WriteLine(staticVariable);
                              }
                            }

        Important Considerations:

      • Variable Shadowing: If you declare a variable with the same name within a nested scope, the inner variable will "shadow" the outer one.
      • Variable Lifetime: A variable's lifetime is the duration for which it exists in memory. Local variables are typically allocated on the stack and destroyed when the method or block ends. Class-level and static variables have a longer lifetime, often the entire duration of the program.
  • Understanding variable scope is crucial for writing well-structured and maintainable C# code. By properly managing the scope of your variables, you can avoid naming conflicts, improve code readability, and prevent unintended side effects.

    • Global Variables

    What are Global Variables in C#?

    • Definition: Global Variables are variables declared outside of any class or method.
    • Accessibility: They are accessible from anywhere within the same file or assembly (a collection of compiled code files).
    • Usage:
      • Rarely used: Due to their potential for introducing side effects and making code harder to maintain, global variables are generally discouraged in modern C# programming.
      • Limited-use cases:
        • Application-wide constants: For values that never change and need to be accessed throughout the application.
        • Singleton pattern: In some cases, global variables can be used to implement the Singleton design pattern, where only one instance of a class exists.

    How to Declare a Global Variable in C#:

    1. Create a separate file (e.g., GlobalVariables.cs): This helps to isolate global variables and improve code organization.
    2. Declare the variable outside of any class:
      3.                         public static class GlobalVariables
                        {
                        public static string AppName = "My Application";
                        public const int MaxItems = 100;
                        }
      • public static: Makes the variable accessible from other parts of the application.
      • const: Defines a constant value that cannot be changed after it's declared.

    Accessing a Global Variable

                        // In another class or method
                    string appName = GlobalVariables.AppName;

    Advantages

    • Side effects: Changes to a global variable can have unintended consequences in other parts of the code.
    • Maintainability: Makes code harder to understand and maintain, especially in large projects.
    • Testing: Can make unit testing more difficult.

    Alternatives to Global Variables

    • Static class properties: A more structured and maintainable way to share data within an application.
    • Dependency injection: A powerful technique for managing dependencies between classes, reducing the need for global variables.
    • Configuration files: For storing application settings in a central location

    Example:

                
                    // GlobalVariables.cs
                    public static class GlobalVariables
                    {
                        public static string ConnectionString = "your_connection_string";
                    }

                    // In another class
                    class MyClass
                    {
                      public void DoSomething()
                      {
                        // Use the connection string
                        using (SqlConnection connection = new SqlConnection(GlobalVariables.ConnectionString))
                        {
                            // ...
                        }
                      }
                    }

    Key Considerations

    • Use with caution: Global variables should be used sparingly and only when absolutely necessary.
    • Consider alternatives: Explore static class proprties, dependency injection, or configuration files before resorting to global variables.
    • Proper naming: Use clear and descriptive names for global variables to improve code readability.

    Constants

    • Definition: Constants are variables whose values cannot be changed after they are initialized. They are declared using the const keyword.
    • Benefits:
      • Readability: Improve code readability by making it clear that certain values should not be modified.
      • Maintainability: Easier to maintain code as you don't need to worry about accidental changes to important values.
      • Performance: In some cases, the compiler can optimize code that uses constants, potentially improving performance.
      • Security: Help prevent unintended modifications to sensitive values.
    • Declaration:
      •                         const int MaxValue = 100;
                        const double Pi = 3.14159;
                        const string CompanyName = "Acme Corp";
    • Data Types: Constants can be of any data type, including:
      • int, long, short, byte, float, double, decimal [numeric types]
      • char, string [character and string types]
      • bool [boolean type]
      • enum [enumeration types]

    • Initialization: Constants must be initialized at the time of declaration
    • Usage:
      •                         int result = MaxValue * 2;
                        double area = Pi * radius * radius;

      Example

                              public class Circle
                        {
                          const double Pi = 3.14159;

                          public double CalculateArea(double radius)
                          {
                            return Pi * radius * radius;
                          }
                        }

      Key Points

      • Constants are declared using the const keyword.
      • Their values cannot be changed after initialization
      • They improve code readability, maintainability, and performance
      • They can be used with any data type

    If...Else Statements

  • Purpose:
    • Used to make decisions in your code based on whether a specific condition is true or false.
    • Control the flow of execution by executing different blocks of code depending on the outcome of the condition
  • Syntax:
    •                         if (condition)
                        {
                          // Code to execute if the condition is true
                        }
                        else
                        {
                          // Code to execute if the condition is false
                        }
  • Explanation
    • Condition:
      • An expression that evaluates to a boolean value (true or false.)
      • Can involve comparisons (e.g. >, <, >=, <=, ==, !=), logical operators (&&, ||, !), and other boolean expressions.
    • True Block:
      • The code within the first set of curly braces {} will execute if the condition is true.
    • False Block(optional):
      • The code wihin the else block will execute if the condition is false.
      • The else block is optional.

    Example:

                            int testScore = 100;
                        if (testScore >= 70)
                        {
                          Console.WriteLine("You have passed this test!");
                        }
                        else
                        {
                          Console.WriteLine("The test was failed.");
                        }
  • Nested if...else Statements:
    • You can nest if...else statements within each other to create more complex decision-making logic.
      •                             if (condition1)
                            {
                                // Code to execute if condition1 is true.
                            }
                            else if (condition2)
                            {
                                // Code to execute if condition1 is false and condition2 is true
                            }
                            else
                            {
                                // Code to execute if both condition1 and condition2 are false
                            }

    Key Points:

    • if...elsestatements are essential for controlling the flow of your C# programs.
    • Use them to make decisions and execute different code paths based on conditions.
    • The else block is optional.
    • You can nest if...else statements for more complex logic.

    While Statements

    Purpose:

    • Used to repeatedly execute a block of code as long as a specific condition remains true
    • Create loops that continue to iterate until the condition becomes false.

    Syntax:

                        while (condition)
                    {
                        // Code to execute as long as the condition is true
                    }

    Explanation:

    • Condition:
      • An expression that evaluates to a boolean value (true or false).
      • Determines whether the loop should continue to execute.
    • Loop Body:
      • The code within the curly braces {} is repeatedly executed as long as the condition remains true.

    Example:

                        int count = 0;
                    while (count < 5)
                    {
                      Console.WriteLine(Count);
                      count++;
                    }
    • This code will print the numbers 0 through 4 to the console.
    • The loop continues to execute as long as the value of count is less than 5.

    Important Notes:

    • Infinite Loops:
      • If the condition is within the whilestatement never becomes false, the loop will execute indefinitely. This can cause your programs to hang or crash.
      • Ensure that there is a mechanism wihin the loop body to eventually make the condition false.
    • do...while Loop:
      • Similar to the while loop, but the condition is checked after the first iteration of the loop body.
      • Guarantees that the loop body will execute at least once
        •                             do
                            {
                              //Code to execute
                            } while (condition);

    Key Points

    • while loops are essential for repetitive tasks in C#.
    • Use them carefully to avoid infinite loops
    • Consider using a do...while loop if you need to ensure that the loop body executes at least once.

    Function Declarations

    • Purpose:
      • Define a reusable block of code that performs a specific task.
      • Break down complex problems into smaller, more manageable units.
      • Improve code organization, readability, and maintainability
    • Syntax:
      •                         return_type function_name(parameter_list)
                        {
                            /* Function body - code to be executed
                            ... */
                            return [value]; // If the function has a return type
                        }
    • Explanation:
      • return_type:
        • Specifies the data type of the value that the function will return.
        • If the function does not return any value, use void.
      • function_name:
        • The name of the function
        • Should be descriptive and meaningful
      • parameter_list:
        • A comma-separated list of parameters that the function accepts.
        • Each parameter consists of a data type and a parameter name
        • If the function doesn't accept any parameters, the parameter list is empty ().
      • function_body:
        • The code that the function executes.
        • Can include any valid C# statements
      • return statement:
        • Used to return a value from the function.
        • The returned value must match the return_type of the function.
        • If the function is void, the return statement is optional.

      Example:

                              int add(int a, int b)
                        {
                          int sum = a + b;
                          return sum;
                        }
      • This function named Add takes two integer parameters (a and b) and returns their sum as an integer.

      Calling a Function:

    • To use a function, you call it by its name and provide the required arguments (if any).
      •                         int result = Add(5, 3); /*Calls the Add function with arguments 5 and 3*/
                        Console.WriteLine(result); /*Output: 8*/

      Key Points:

      • Functions are fundamental building blocks of C# programs.
      • They promote code reusability and improve code organization
      • Use meaningful function names to enhance code readability
      • Choose appropriate return types for your functions.
